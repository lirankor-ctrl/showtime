# 🎭 SHOW TIME — יומן התרבות שלי

אפליקציית PWA בעברית (RTL) למעקב, תכנון ותיעוד אירועי תרבות: הצגות, הופעות, סרטים, קונצרטים, סטנדאפ, הרצאות, פסטיבלים ועוד.

הנתונים נשמרים באופן מאובטח בחשבון אישי ב-**Supabase** (התחברות, מסד נתונים ואחסון תמונות). כל משתמש רואה ועורך אך ורק את הנתונים שלו, באמצעות Row Level Security.

## יכולות

- **דף הבית** — האירוע הקרוב מוצג בגדול, ומתחתיו רשימת האירועים הקרובים לפי תאריך.
- **ניהול אירועים** — יצירה, עריכה ומחיקה עם כל הפרטים (קטגוריה, מקום, מקומות ישיבה, כרטיסים, מחיר, קישור, הערות).
- **עמוד אירוע** — תכנון לפני, וארכיון זיכרונות אחרי: דירוג 1–5, סקירה אישית, רגעים אהובים ותמונות.
- **לוח שנה** — תצוגה חודשית עם נקודות צבע לפי קטגוריה.
- **מנויים** — מעקב אחרי כרטיסים שנותרו, התראות על מנוי שמסתיים, וקישור אוטומטי לאירועים.
- **שיתוף** — Web Share API מקומי, עם נפילה לאחור להעתקת סיכום בעברית ללוח.
- **התראות מקומיות** — תזכורת בבוקר של יום האירוע (ראו הערה למטה).
- **גיבוי** — ייצוא/ייבוא של כל הנתונים כקובץ JSON.
- **PWA** — ניתן להתקנה במסך הבית באייפון ובאנדרואיד.

## הגדרת Supabase

1. צרו פרויקט ב-[Supabase](https://supabase.com).
2. הריצו את `supabase/schema.sql` ב-SQL editor (יוצר טבלאות, RLS, מדיניות, bucket פרטי `event-photos`, אינדקסים וטריגרים).
3. העתיקו `.env.example` ל-`.env.local` ומלאו:

```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

> השתמשו אך ורק במפתח ה-`anon` הציבורי. אסור לחשוף את ה-`service_role` בקוד הדפדפן — האבטחה נאכפת על ידי RLS.

## הרצה

```bash
npm install
npm run dev        # פיתוח
npm run build      # בנייה לפרודקשן
npm run preview    # תצוגה מקדימה של הבנייה (כולל PWA)
```

מי שכבר השתמש בגרסה המקומית — לאחר ההתחברות הראשונה תוצע העברה חד-פעמית של הנתונים המקומיים לחשבון.

> התראות מתוזמנות כשהאפליקציה **סגורה לחלוטין** דורשות שרת (Web Push). בשלב המקומי נתמכת תזכורת בעת פתיחת האפליקציה ביום האירוע. פירוט מלא ב־`src/utils/notifications.ts`.

לפרטי ארכיטקטורה למפתחים — ראו [`CLAUDE.md`](./CLAUDE.md).
